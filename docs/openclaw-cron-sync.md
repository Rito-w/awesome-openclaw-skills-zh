# Awesome OpenClaw Skills 翻译同步 - 定时任务方案

## 📋 项目说明

- **本地仓库**: `/Volumes/myDisk/workplace/awesome-openclaw-skills-zh`
- **上游仓库**: `https://github.com/VoltAgent/awesome-openclaw-skills.git`
- **目标**: 每隔一天检查上游更新，有更新时派发翻译任务给 Claude Code

---

## ⚠️ 关键限制

`sessions_spawn` **必须在 OpenClaw 会话内运行**，无法通过 cron 直接调用！

因此采用 **定时检查 + 通知用户手动派发** 的方案。

## 🛠️ 管理命令

```bash
# 查看所有定时任务
openclaw cron list

# 手动触发测试（立即执行）
openclaw cron run 4de5b07c-7668-4234-9033-913a99b95762 --force

# 查看运行历史
openclaw cron runs --id 4de5b07c-7668-4234-9033-913a99b95762 --limit 10

# 修改执行频率（比如改为每周一、三、五）
openclaw cron edit 4de5b07c-7668-4234-9033-913a99b95762 --cron "0 10 * * 1,3,5"

# 删除任务
openclaw cron remove 4de5b07c-7668-4234-9033-913a99b95762
```

## 📝 用户操作指南

### 收到更新通知后

1. **生成翻译任务提示词**
   ```bash
   cd /Volumes/myDisk/workplace/awesome-openclaw-skills-zh
   python3 scripts/auto-dispatch.py --next 5
   ```

2. **复制输出的提示词**

3. **在 OpenClaw 会话中派发任务**
   ```
   sessions_spawn(task="""...复制的提示词...""", label="translate-1")
   ```

4. **等待完成通知**
   - 子代理完成后会自动发送 System Message

5. **提交并推送**
   ```bash
   git add -A
   git commit -m "feat: 翻译更新 (YYYY-MM-DD)"
   git push
   ```

## 🔔 替代方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **OpenClaw Cron + 手动派发** | 灵活可控，翻译质量高 | 需要手动操作 |
| **GitHub Actions + Issue 通知** | 完全自动检测，不错过更新 | 仍需手动翻译 |
| **完全手动** | 最简单，无需配置 | 容易忘记检查 |

**推荐**: 同时启用 OpenClaw Cron 和 GitHub Watch，双重保障不错过更新。
